## 1. Project Overview

### Project Name
**Teta Lo2is** (Arabic: تيتا لوئيس — "Grandma Lo2is")

### Main Purpose
An **Arabic-language educational car racing game** for children. Players drive a car down a 3D perspective road, dodge enemy vehicles, and must periodically refuel by answering multiple-choice questions related to Biblical Old Testament stories.

### Business Goal
To make learning Bible stories engaging and interactive for Arabic-speaking children, wrapping quiz content inside a car racing game mechanic. The educational theme is the Old Testament narrative (Creation → Adam & Eve → Cain & Abel → Noah's Ark → Tower of Babel → Abraham).

### Target Users
- Arabic-speaking children (ages approximately 6–12)
- Parents/teachers using it as an educational tool
- Church Sunday-school contexts (implied by the content)

### Main Features
| Feature | Description |
|---|---|
| 6 progressive levels | Each level covers one Biblical story, with increasing difficulty |
| Car racing gameplay | 3-lane perspective road with enemy cars to dodge |
| Fuel-based survival | Fuel drains over time; reaching zero = game over |
| Gas Station quiz | Mid-race Q&A modal: 3 Arabic questions with 4 colored answers each |
| Story video intro | YouTube embedded video plays before each race |
| Audio support | Questions are voiced (MP3); game has music and SFX |
| Leaderboard | Firebase Firestore-backed global leaderboard (top 10 by total score) |
| User auth | Firebase Auth (email/password) with registration and password reset |
| Progress persistence | Stars and unlocked levels saved in Firestore (for logged-in users) |
| Guest mode | Fully playable without login; progress only stored in component state |
| Confetti win animation | canvas-confetti fireworks on level completion |

### Supported Platforms
- Web browser (desktop + mobile; touch-swipe supported)
- Deployed to **GitHub Pages** at `https://kiroadel14.github.io/teta_lo2is/`

### Technology Stack

| Category | Technology |
|---|---|
| Framework | React 18.3.1 + Vite 8 |
| Language | TypeScript (TSX) |
| Routing | React Router v7 (installed but **not actively used** — screen state is managed manually) |
| Styling | Tailwind CSS v4 (via `@tailwindcss/vite`) + inline styles |
| Animations | Framer Motion (`motion` package v12) |
| UI Components | Radix UI (full suite, shadcn-style wrappers in `ui/`) |
| Icons | Lucide React + MUI Icons |
| Backend/DB | Firebase v12 (Auth + Firestore) |
| Build/Deploy | Vite build → `gh-pages` npm package → GitHub Pages |
| Package Manager | npm (pnpm workspace file also present but not primary) |
| Fonts | Google Fonts: Cairo (Arabic), Fredoka One |

### Plain English Explanation

> A child opens the game, optionally logs in, picks a Biblical story level, watches a short YouTube explainer video, then races a car down a road dodging enemy cars. Partway through the race the car needs "fuel" — a gas station pop-up appears with 3 multiple-choice questions about the story they just watched. Answering correctly refills fuel. Surviving the full race distance wins the level, earns stars, unlocks the next story, and saves the score to a cloud leaderboard.

## 3. Architecture Analysis

### Architecture Pattern
**Single Page Application (SPA)** with a **manual screen state machine** in the root `App.tsx`. There is no client-side router (React Router is installed but unused). All navigation is done by flipping a `screen` state variable.

### Screen State Machine

```
Screen type:
'title' | 'auth' | 'levelSelect' | 'story' | 'racing' | 'results' | 'leaderboard'

Transition diagram:

  [title] ──onStart──► [levelSelect]
  [title] ──onLogin──► [auth]
  [title] ──onShowLeaderboard──► [leaderboard]
  [leaderboard] ──onBack──► [title]
  [auth] ──onLoginSuccess──► [levelSelect]
  [auth] ──onBack──► [title]
  [levelSelect] ──onSelectLevel──► [story]
  [levelSelect] ──onBack──► [title]
  [story] ──onPlay──► [racing]
  [story] ──onBack──► [levelSelect]
  [racing] ──onGameOver(won)──► [results]
  [racing] ──onGameOver(lost)──► [results]
  [racing] ──onBack (via pause)──► [levelSelect]
  [results] ──onRetry──► [racing]
  [results] ──onBack──► [levelSelect]
```

### State Management

All global state lives in `App.tsx` using React `useState` hooks. **No external state manager** (Redux, Zustand, etc.) is used.

| State Variable | Type | Purpose |
|---|---|---|
| `user` | `any` (Firebase User) | Currently authenticated user |
| `loading` | `boolean` | Show loading screen while Firebase Auth initializes |
| `screen` | `Screen` | Which screen is currently visible |
| `selectedLevelId` | `string` | ID of the level currently selected (default: `'level_1'`) |
| `gameResult` | `GameResult \| null` | `{won, score, stars}` result from last race |
| `levelStars` | `Record<string, number>` | Best star rating per level |
| `levelScores` | `Record<string, number>` | Best score per level |
| `leaderboardData` | `any[]` | Top-10 players from Firestore |
| `raceKey` | `number` | Incremented to force-remount `RaceScreen` on retry |
| `unlockedLevelIds` | `Record<string, boolean>` | Which levels are unlocked (always includes `level_1`) |

### Component Hierarchy

```
App.tsx
├── TitleScreen (inline function in App.tsx)
├── AuthScreen.tsx
├── LevelSelect.tsx
├── StoryVideoScreen.tsx
├── RaceScreen.tsx
│   └── GasStationModal.tsx
└── ResultsScreen.tsx
```

All transitions are wrapped with `<AnimatePresence>` + `<motion.div>` from Framer Motion for smooth enter/exit animations.

### Firebase Architecture

```
Firebase Project: teta-lo2is
  ├── Authentication: Email/Password provider
  └── Firestore Database
      └── Collection: "users"
          └── Document: {uid}
              ├── displayName: string
              ├── email: string
              ├── totalScore: number         ← sum of best scores across all levels
              ├── unlockedLevels: string[]   ← e.g. ["level_1", "level_2"]

              ├── scores: { level_1: 1200, ...} ← best score per level

```

## 5. Game Engine & Core Systems

### 5.1 Road Rendering System (SVG-based 3D Perspective)

The road is rendered as a full-screen `<svg viewBox="0 0 100 100">` with `preserveAspectRatio="xMidYMid slice"`.

**Key Constants:**
```
VP_X = 50              // Vanishing point X (center)
HORIZON_Y = 33         // Y of horizon line (33% from top = 33% sky)
ROAD_BOTTOM = 100      // Bottom of viewbox
ROAD_HALF_BOTTOM = 42  // Road half-width at viewer's feet (x: 8–92)
ROAD_HALF_HORIZON = 6.5 // Road half-width at horizon (x: 43.5–56.5)
```

**Perspective Math:**
```typescript
function perspX(bottomOffset: number, t: number): number {
  const ratio = ROAD_HALF_HORIZON / ROAD_HALF_BOTTOM;
  const horizonOffset = bottomOffset * ratio;
  const ease = Math.pow(t, 0.65); // exponential easing
  return horizonOffset + (bottomOffset - horizonOffset) * ease;
}
// t=0: at horizon (small), t=1: at viewer's feet (large)
```

**3 Lanes:**
- Left lane center: `VP_X - 21 = 29`
- Center lane center: `VP_X = 50`
- Right lane center: `VP_X + 21 = 71`

**SVG Layer Stack (top to bottom, rendering order):**

| Layer | Content |
|---|---|
| 1 | Sky gradient rectangle |
| 2 | Distant mountains (2 polygon layers + snow caps) |
| 3 | Near hills (ellipses) |
| 4 | Background tree line silhouette (pine triangles) |
| 5 | Ground / grass base |
| 6 | Road (crest above horizon + main trapezoid) |
| 7 | Lane markings (dashed white lines, scrollable) |
| 8 | Roadside bollards (white posts with orange tops, animated scroll) |
| 9 | Roadside trees (left + right banks, pine + round deciduous) |
| 10 | Horizon atmospheric mist overlay |
| 11 | Clouds (4 volumetric multi-ellipse clouds) |
| 12 | Enemy car sprites (perspective-scaled PNG images) |
| 13 | Player car sprite (Framer Motion animated position) |
| 14 | Red vignette (low/critical fuel warning) |
| 15 | Collision flash (solid red overlay, 600ms) |

### 5.2 Game Loop

Implemented using `requestAnimationFrame` inside a `useEffect` with refs for performance-critical state.

**Loop runs ~60fps. Each frame:**

```
1. Compute deltaSeconds (capped at 0.05 to avoid spiral-of-death on tab blur)
2. Advance distanceRef += DISTANCE_SPEED * deltaSeconds
3. Drain fuelRef -= FUEL_DRAIN_PER_SECOND * deltaSeconds
4. Advance scrollOffset (drives lane marking animation)
5. Increment spawnFrameCounter += deltaSeconds * 60
6. If spawnFrameCounter >= level.spawnRate:
   - Spawn new obstacle at random lane (0, 1, or 2)
   - Assign random enemy car sprite (0, 1, or 2)
   - Reset spawn counter
7. Advance all obstacles: obs.t += level.obstacleSpeed * deltaSeconds * 60
8. Remove obstacles where obs.t >= 1.15 (past viewer)
9. Collision detection: find obstacle in playerLane where 0.88 < t < 1.05
   - If found AND no cooldown: play crash sound, -20 fuel, 90-frame cooldown, flash
10. Gas station trigger: if !visited && distanceProgress >= 45% && fuel <= 50%
    → open GasStationModal
11. Game over check: if fuel <= 0 → play game-over sound, compute stars, call onGameOver(won=false)
12. Win check: if distance >= target AND fuel > 0 → play winner sound, compute stars, call onGameOver(won=true)
13. Sync refs → setState (fuel, score, obstacles) for HUD re-render
```

**Loop pauses when:** `pausedRef.current === true` OR `showGasStationRef.current === true`

---

### 5.3 Fuel System

```
Initial fuel: 100%
Target race duration: 90 seconds
Fuel drain rate: 100 / (90 * 1.15) ≈ 0.966% per second
  → At 90s, fuel would be ~13% remaining if no collisions
  → Gas station adds fuel at ~45% progress

Collision penalty: -20% per collision (90-frame / ~1.5s cooldown)

Fuel states:
  normal:   fuel >= 25%  → green bar
  low:      fuel < 25%   → amber bar + pulsing emoji + warning banner
  critical: fuel < 5%    → red bar + red vignette overlay + urgent banner

Gas station trigger: distanceProgress >= 0.45 AND fuel <= 50
  (triggers at most ONCE per race — gasStationVisitedRef guards it)

Fuel refill per correct answer: +25% (capped at 100%)
  → 3 correct = +75% fuel
  → 2 correct = +50% fuel
  → 1 correct = +25% fuel
  → 0 correct = 0 fuel added
```

---

### 5.4 Scoring System

```
score = Math.floor(distanceRef.current)
  → Score IS the distance traveled in arbitrary units

Win stars:
  Won:
    fuel >= 60% at finish → 3 stars
    fuel >= 30% at finish → 2 stars
    fuel > 0% at finish  → 1 star

  Lost (fuel = 0 before finish):
    score >= survivalTargetDistance     → 3 stars (impossible case actually)
    score >= survivalTargetDistance*0.6 → 2 stars
    score >= survivalTargetDistance*0.3 → 1 star
    else                                → 0 stars

Best scores/stars: Math.max(current, new) — never regresses
totalScore: sum of all level best scores (used for leaderboard ranking)
```

### 5.5 Obstacle System

```typescript
interface Obstacle {
  id: number;       // auto-incrementing ID
  lane: number;     // 0=left, 1=center, 2=right
  t: number;        // depth parameter: 0=far (horizon), 1=at player
  color: string;    // unused (was for colored boxes, now uses PNG sprites)
  colorDark: string; // unused
  spriteIndex: number; // 0, 1, or 2 → selects ENEMY_CARS[spriteIndex]
}

Spawn timing: spawnFrameCounter accumulates at 60fps equivalent
  → When counter >= level.spawnRate, spawn new obstacle, reset counter

Obstacle movement: t += level.obstacleSpeed * deltaSeconds * 60
  → At t >= 1.15, obstacle is removed (past viewer)
  → Collision window: 0.88 < t < 1.05 (just in front of player)

Rendering: perspX/laneXAtT functions place car at correct screen position
  → Size: s = obs.t * 6 (appears tiny at horizon, large near player)
  → Minimum render size: s >= 0.5
```

## 8. Authentication Flow

```
App Startup
  │
  ├─ onAuthStateChanged fires
  │   ├─ user exists → fetch Firestore doc → populate levelStars, levelScores, unlockedLevels
  │   └─ user is null → reset to defaults (level_1 only)
  │
  └─ Show TitleScreen

Login Flow:
  TitleScreen → [Login button] → AuthScreen (isLoginMode=true)
    → Fill email + password → Submit
    → signInWithEmailAndPassword
    → onLoginSuccess() → screen = 'levelSelect'
    → App useEffect detects new user → fetches Firestore data

Register Flow:
  AuthScreen (isLoginMode=false)
    → Fill firstName, lastName, email, password → Submit
    → createUserWithEmailAndPassword
    → updateProfile (set displayName)
    → setDoc (create Firestore doc with defaults)
    → 1s delay → onLoginSuccess() → screen = 'levelSelect'

Guest Flow:
  TitleScreen → [Start Game button] (without logging in)
    → screen = 'levelSelect'
    → Progress stored in React state only (lost on page refresh)

Logout:
  TitleScreen → [Logout button] → signOut(auth)
    → onAuthStateChanged fires (user=null)
    → Reset levelStars, levelScores, unlockedLevelIds to defaults
```

## 9. User Flows

### 9.1 Complete Game Flow

```
1. User opens app
   └─ Loading screen shown while Firebase Auth initializes
   └─ TitleScreen appears

2. User taps "ابدأ اللعب" (Start Game)
   └─ Screen = 'levelSelect'
   └─ Level grid shown (only level_1 unlocked initially)

3. User taps Level 1
   └─ Screen = 'story'
   └─ YouTube video autoplays (level 1: Creation story)

4. Video ends
   └─ "يلا نلعب!" button appears
   └─ User taps it → screen = 'racing', raceKey incremented

5. Racing starts
   ├─ fuel = 100%, score = 0, 90-second target
   ├─ Player dodges enemy cars (left/right arrow or swipe)
   ├─ At ~45% distance AND fuel ≤ 50%:
   │   └─ GasStationModal slides up
   │   └─ 3 questions answered (audio + 4 color buttons)
   │   └─ +25% fuel per correct answer
   │   └─ 3-2-1 countdown → modal closes → racing resumes
   └─ Two outcomes:
       ├─ FUEL = 0: Game over → screen = 'results' (won=false)
       └─ DISTANCE COMPLETE + fuel > 0: Win → screen = 'results' (won=true)

6. Results screen
   ├─ Won: confetti, 🏆, stars, "أحسنت يا بطل!"
   └─ Lost: 🚗💨, "خلصت بنزين!"

7. If won (logged in):
   └─ Firestore updated (stars, scores, totalScore, unlockedLevels)
   └─ Next level unlocked

8. Retry:
   └─ raceKey++ → RaceScreen remounted fresh
   └─ screen = 'racing'
```

## 15. AI_AGENT_CONTEXT

### Core Project Summary

**Teta Lo2is** is an Arabic educational car-racing game for children built with React 18 + Vite. Players race a car on a 3D SVG perspective road, dodge enemy vehicles, and answer Bible story questions at a mid-race gas station to refuel. There are 6 levels covering Genesis stories. The game uses Firebase Auth + Firestore for user accounts and a global leaderboard. Deployed to GitHub Pages.

---

### Architecture Summary

- **No routing library in use** — navigation is a `screen` state string in `App.tsx`
- **No state manager** — `useState` + `useRef` only, all in `App.tsx`
- **Game loop runs in `requestAnimationFrame`** — performance-critical values use refs, not state
- **SVG-based game rendering** — not Canvas, not WebGL — pure SVG with perspective math
- **Firebase is the only "backend"** — Auth + Firestore; no custom server

### Critical Workflows

**Adding a new level:**
1. Add entry to `LEVELS` array in `src/app/data/levels.ts`
2. Add 3 question audio files to `/public/audio/levels/levelN/question[1-3].mp3`
3. Add YouTube embed URL as `videoUrl`
4. Level is auto-added to the grid in `LevelSelect.tsx` (grid is `3×N`, currently `3×2`)

**Changing game difficulty:**
1. Edit `survivalTargetDistance`, `obstacleSpeed`, `spawnRate` in the level object in `levels.ts`

**Changing fuel/scoring logic:**
1. Edit `RaceScreen.tsx`: `FUEL_DRAIN_PER_SECOND`, `TARGET_DURATION_SECONDS`, collision penalty (`-20`), star thresholds

**Adding a new screen:**
1. Add to the `Screen` type union in `App.tsx`
2. Add conditional render inside `<AnimatePresence>` in the JSX
3. Wire navigation in/out via callback props

**Deploying:**
```bash
npm run deploy   # runs build then gh-pages -d dist
```

### Key Business Rules

1. Level 1 is always unlocked; all others start locked
2. Levels unlock sequentially — completing level N unlocks level N+1
3. Stars and scores always record the **best** (never regress)
4. `totalScore` is the **sum of all level best scores** — used for leaderboard ranking
5. Gas station triggers **at most once per race**, at ≥ 45% progress AND ≤ 50% fuel
6. Each correct gas station answer adds exactly **+25% fuel**
7. Guest users get no cloud save — progress is in-memory only
8. The game is fully **RTL (right-to-left)** — `dir="rtl"` on all screens

### Things That Must Not Be Broken

1. **`base: "/teta_lo2is/"` in `vite.config.ts`** — removing this breaks GitHub Pages deployment
2. **`gameOverFiredRef`** — prevents double-calling `onGameOver`. Do not remove.
3. **`gasStationVisitedRef`** — prevents infinite quiz loops. Do not remove.
4. **`raceKey` increment pattern** — `setRaceKey(k => k+1)` causes RaceScreen remount on retry. This is how the game resets — do not use component reset without this.
5. **`collisionCooldownRef`** — prevents instant fuel drain from one obstacle. Required for fair gameplay.
6. **`showGasStationRef`** — the game loop checks this ref (not state) to pause physics. Both ref and state must be updated together in `handleGasStationComplete`.
7. **Audio cleanup in RaceScreen** — `mainLoopRef.current?.pause()` in the useEffect cleanup. Without this, music keeps playing on screen change.
